# 🎬 视频资源下载器 - Edge 浏览器扩展

抓取当前页面可下载的视频资源列表，支持指定下载路径。

## 功能特性

- **DOM 扫描**：检测页面中 `<video>`、`<source>`、视频链接等
- **网络拦截**：通过 `webRequest` API 拦截正在请求的视频 URL（可捕获动态加载的视频）
- **指定下载路径**：保存到 `Downloads/你指定的子文件夹` 下
- **批量复制链接**：支持一键复制视频 URL
- **HLS/DASH 检测**：标注 HLS (.m3u8) 和 DASH (.mpd) 流媒体

## 安装方法

### 1. 加载扩展（开发者模式）

1. 打开 **Microsoft Edge**，地址栏访问 `edge://extensions/`
2. 右上角开启 **「开发者模式」**
3. 点击 **「加载压缩的扩展」**
4. 选择本项目文件夹（包含 `manifest.json` 的目录）
5. 扩展图标将出现在工具栏

### 2. 使用方法

1. 打开任意包含视频的网页
2. 点击扩展图标，弹出面板
3. 等待扫描完成，查看视频列表
4. 设置下载路径（子文件夹名称，如 `videos/2026`）
5. 点击「下载」按钮开始下载

## 下载路径说明

出于安全原因，浏览器扩展**无法**写入任意绝对路径（如 `D:\Videos`）。

| 设置方式 | 效果 |
|---------|------|
| 留空 | 直接保存到系统默认下载文件夹 |
| `my-videos` | 保存到 `Downloads/my-videos/` |
| `videos/2026/04` | 保存到 `Downloads/videos/2026/04/` |

勾选「另存」则每次下载都弹出系统另存为对话框，可自行选择位置。

## 支持的视频格式

| 格式 | 可直接下载 | 需特殊处理 |
|------|-----------|-----------|
| .mp4 / .webm / .ogg | ✅ | - |
| .m3u8 (HLS) | ❌ | 需 ffmpeg 合并 |
| .mpd (DASH) | ❌ | 需 yt-dlp 等工具 |
| Blob URL | ❌ | 需 MediaRecorder 录制 |
| 加密/DRM 视频 | ❌ | 无法下载 |

## 网络拦截说明

**为什么有些视频没有检测到？**

- 扩展使用了 `webRequest` API 拦截网络请求
- 视频必须**开始请求后**才会被拦截
- 如果视频 URL 是页面加载后才生成的（如点击播放按钮），需要**先触发播放/加载**，然后再点击扩展扫描

**优化扫描步骤：**
1. 打开页面 → 不要立即打开扩展
2. 如果视频是懒加载，先滚动到视频位置 / 点击播放
3. 等视频开始缓冲（看到进度条动）→ 再打开扩展
4. 点击「刷新」按钮重新扫描

## HLS / DASH 流媒体处理（进阶）

对于 .m3u8 (HLS) 和 .mpd (DASH) 视频，需要用命令行工具下载：

```bash
# HLS 下载
ffmpeg -i "视频URL" -c copy output.mp4

# 或用 yt-dlp（推荐，支持更多格式）
yt-dlp "视频URL"
```

## 技术限制

1. **HLS/DASH/Blob** 视频无法通过扩展直接下载，需要第三方工具
2. **DRM 加密**视频（如 Netflix、Disney+）无法解密下载
3. 扩展需要 `<all_urls>` 权限才能拦截所有网站的视频请求
4. Edge/Chrome 的 `downloads` API 不支持任意绝对路径

## 常见问题

**Q: 为什么下载按钮是灰色的？**
A: 该视频是 HLS/DASH/Blob 流媒体，无法直接通过 URL 下载。

**Q: 下载路径填了但没生效？**
A: 确认填的是**子文件夹名称**，不是绝对路径。如果勾选了「另存」，会弹出系统对话框优先。

**Q: 有些视频没被检测到？**
A: 尝试先让视频开始加载/播放，然后再刷新扩展的扫描。
